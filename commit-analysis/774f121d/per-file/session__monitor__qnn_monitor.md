# src/winml/modelkit/session/monitor/qnn_monitor.py

## TL;DR
The single biggest file in the v2.9 squash. A stub `QNNMonitor(EPMonitor)` (47 lines, two no-op methods, `is_available()→False`) is replaced wholesale by a 657-line concrete `WinMLEPMonitor` subclass that owns: (a) the QNN EP profiling provider-option contract (C-3 — `profiling_level` and `profiling_file_path` are owner-enforced after merge), (b) the FR-14 four-layer op-type fallback chain (ONNX → EP-authoritative → heuristic → raw), (c) the CSV + QHAS artifact-parsing pipeline with the Windows file-handle-lag retry (R-2), (d) a `parse_existing_artifacts` offline path, and (e) the `_to_int(round(float(...)))` hardening that prevents float-string SDK output from silently zeroing every `duration_us`. The fallback chain is wired correctly — verified end-to-end. The old `QNNProfiler.run(iterations, warmup)` loop and `_working_directory`/`_generate_inputs` helpers are intentionally gone: those concerns moved to `WinMLSession.perf()` (loop) and were dropped (CWD switching — replaced by mtime-gated glob; input generation — caller's job now).

## Diff metrics
- Lines added / removed: **+626 / -23** (657 lines at HEAD)
- New / modified: **modified** — same path existed as a 47-line stub before this squash; the squash also deletes `optracing/qnn/profiler.py` (-351 lines), `optracing/qnn/csv_parser.py` (-227), `optracing/qnn/qhas_parser.py` (-113), and adds the new home at `session/monitor/qnn/_internal.py` (+443) and moves `viewer.py`.
- The pre-image (`5a4fb95c`) was a placeholder: `__enter__/__exit__/is_available` were all explicit no-ops. There is no "earlier real version" of this file at this path — the squash replaces the stub.

## Role before vs after
- **Before (`5a4fb95c`)**: Forward-looking placeholder. Inherited `EPMonitor` (the v2.3-era base name), declared `is_available()→False`, and had a comment promising "future QAIRT profiling via qnn-profile-viewer.exe." None of the real op-tracing logic lived here; the working implementation was at `src/winml/modelkit/optracing/qnn/profiler.py` (`QNNProfiler(OpTracer)`) which owned its own ORT session, input generation, warmup/iteration loop, CWD switching, and CSV/QHAS collection inside a single `run(iterations, warmup)` method.
- **After (`692d08fa`)**: The single source of truth for QNN op-tracing. Inherits the v2.4-renamed `WinMLEPMonitor`. Lifecycle inverts: instead of `QNNProfiler.run()` driving its own session, `WinMLSession.perf(monitor=QNNMonitor(level))` is the entry point — QNNMonitor contributes session/provider options via two hooks (`get_session_options`, `get_provider_options`), and parses artifacts post-hoc inside `__exit__`. ClassVars (`ep_name="qnn"`, `requires_session_teardown=True`) encode the EP-pinning and CSV-flush-on-destroy invariants as part of the type, not as scattered runtime checks.

## Symbol-level changes
All deltas are additions on top of the stub:

- **Module constant `_LEVEL_TO_PROFILING: dict[str, str]`** — `{"basic": "detailed", "detail": "optrace"}`. The user-facing "basic"/"detail" vocabulary does not match QNN EP's `profiling_level` strings; this map is the only thing translating them. Note the counterintuitive `"basic" → "detailed"`.
- **Class `QNNMonitor(WinMLEPMonitor)`**:
  - **`requires_session_teardown: ClassVar[bool] = True`** — encodes C-2: QNN EP flushes the profiling CSV only on `InferenceSession` destruction, so `WinMLSession.perf().__exit__` must `self.reset()` **before** calling `monitor.__exit__`. Validated against session.py:732: `if getattr(effective_monitor, "requires_session_teardown", False): self.reset()` precedes `effective_monitor.__exit__(...)`.
  - **`ep_name: ClassVar[str | None] = "qnn"`** — pins `WinMLSession` to the QNN EP path so provider options flow through `add_provider_for_devices`. Without this, sessions without an explicit `ep` fall back to ORT's policy-based selection which silently drops provider options (and would render the entire monitor inert).
- **`__init__(level, output_dir=None, extra_provider_options=None)`**:
  - Validates `level in _LEVEL_TO_PROFILING` up front (replaces silent fallback in old code).
  - When `output_dir is None`, mints a fresh `tempfile.mkdtemp(prefix="qnn_profile_")`. **Deliberately does NOT register a finalizer** — artifacts persist for inspection (documented in class + property docstrings; called out in the OQ section as a leak risk).
  - `_csv_path` is **resolved at `__init__`**, satisfying NFR-4 (file paths produced at `__init__`, not per-call).
  - `_extra` is a defensive `dict(extra_provider_options or {})` copy.
  - `_onnx_op_types: dict[str, str] = {}` initialized empty — primed later by `set_onnx_op_types`.
- **`output_dir` property** — read-only accessor for the artifact directory.
- **`is_available()` classmethod** — two-path probe with the **Batch D behaviour change**:
  - Path 1: `onnxruntime-qnn` bundled wheel → `QNNExecutionProvider in ort.get_available_providers()`.
  - Path 2: `onnxruntime-windowsml` → `WinMLEPRegistry.instance()` (filesystem-only discovery; **no eager DLL loading** per the squash's lazy-load contract — this is the rename from `ensure_initialized()` to `.instance()`), then look for QNN device in `ort.get_ep_devices()`.
  - WARNING-log on any `Exception` (NFR-2: no silent failures). Returns `False` after logging.
  - Inline TODO acknowledges that if `ort.get_ep_devices()` ever requires a loaded DLL for visibility, the probe will need to call `register_ep` on the matching `EPEntry` directly.
- **`get_session_options()`** — returns `{"ep.context_enable": "1", "ep.context_embed_mode": "0"}`. The docstring is explicit that `session.disable_cpu_ep_fallback` is **intentionally NOT** set: under `onnxruntime-windowsml` the WinML-registered QNN partitions QDQ-wrapped EPContext models into Q/DQ-on-CPU + EPContext-on-QNN, which is correct; disabling CPU fallback would reject that valid partition. This is a deliberate divergence from the old `QNNProfiler._build_session_options` which set it.
- **`get_provider_options()`** — C-3 invariant: `dict(self._extra)` first, then `opts["profiling_level"]` and `opts["profiling_file_path"]` assigned after the merge so caller cannot shadow them. Docstring justifies why the bundled-ORT defaults (`backend_path=QnnHtp.dll`, `htp_performance_mode=high_performance`, `htp_graph_finalization_optimization_mode=3`, `enable_htp_fp16_precision=1`) are no longer hardcoded: they would overwrite WinML's pre-tuned absolute `backend_path` and break DLL loading. **Behavioural change: those four HTP defaults must now be supplied by the caller via `extra_provider_options` if running the bundled-ORT path.**
- **`__enter__` / `__exit__`**:
  - `__enter__` adds a re-entry guard (`raise RuntimeError("QNNMonitor already entered")`) absent from the old code.
  - `__exit__` routes through `_parse_artifacts_safe`, never suppresses caller exceptions (implicit `None` return).
- **`_parse_artifacts_safe(qhas_override=None)`** — single source of truth for parse-failure conversion. Both `__exit__` (live) and `parse_existing_artifacts` (offline) route through it so they cannot diverge; both produce `OpTraceResult(status="parse_failed", error=str(exc))` on exception.
- **`result` property** — accessor for the parsed `OpTraceResult` (concrete override of `WinMLEPMonitor.result`'s default `getattr(self, "_result", None)`).
- **`set_onnx_op_types(onnx_op_types)`** — overrides the no-op default; defensively copies the input so post-injection mutation by the caller can't corrupt L1. Empty input is a valid no-op.
- **`_resolve_op_type(op_path, ep_authoritative=None)`** — the FR-14 chain. **Walked carefully below** in the critical-check section.
- **`_heuristic_op_type(op_path)`** — `_TOKEN_SUFFIX.sub("", op_path).strip()` then `rsplit("/", 1)[-1].strip()`, with trailing-slash fallback to the cleaned input so empty strings never propagate.
- **`parse_existing_artifacts(level, artifacts, onnx_op_types=None)` classmethod** — offline entry point. Requires `"csv"` key; raises `ValueError` if absent. Routes through `_parse_artifacts_safe` so corrupt artifacts surface as `parse_failed` instead of propagating. **M-2 carry-forward**: explicitly sets `instance._result` so callers holding the wrapper instance see a populated `result` property. Reaches into `instance._csv_path` to overwrite the constructor-pinned default with the caller's explicit path.
- **`_parse_artifacts(qhas_override=None)`** — the meat:
  - R-2: file-missing → `time.sleep(0.05)` → re-check → `no_data` if still absent.
  - Calls `parse_qnn_profiling_csv(csv_path)` (from `qnn._internal`).
  - **Inline closure `_to_int(val, field)` — the hardening**: `round(float(val))` instead of `int(val)`. Catches `TypeError`/`ValueError`, logs WARNING, returns 0. Used for `accel_execute_cycles` and `accel_execute_us`.
  - Builds `OperatorMetrics` from CSV rows; resolves op-type via L1+L3+L4 (`ep_authoritative=None`).
  - Detail mode: calls `_try_qhas`; on success replaces operators+summary; on failure sets `status="basic_fallback"` and keeps CSV data.
  - **`num_samples=int(meta.get("num_samples", 0) or 0)`** — not routed through `_to_int`. Latent fragility if QNN ever emits `"5.0"` here, but `_TOKEN_SUFFIX`-style integer counts are unlikely to be floats in practice.
- **`_try_qhas(artifacts, qhas_override=None)`** — returns `(summary, operators, qhas_path)` or `(None, None, None)`. Per C-5/FR-12 **never calls `os.chdir`**. Offline path: parse `qhas_override` directly. Live path: locate `*_qnn.log` + schematic (via `_find_schematic`), find SDK, shell out to `run_qhas_viewer`. QHAS-built `OperatorMetrics` carry `samples_us=[op["duration_us"]]` (single-snapshot fanout for downstream p90 consistency).
- **`_find_schematic()`** — output_dir glob first, **mtime-gated** CWD fallback: `p.stat().st_mtime >= csv_mtime - 5.0`. The 5-second tolerance is the filesystem-clock-skew slop. Closes the silent-corruption hole where a stale schematic from a prior CI run would be consumed and produce QHAS metrics for the wrong graph with `status="ok"`.
- **`_make_failure_result(status, error)`** — minimal `OpTraceResult` factory; always includes the CSV path in artifacts (even when no data was produced — useful for forensics).

## Behavior / contract changes

- **Loop ownership inversion (FR-2)**: `QNNProfiler.run(iterations, warmup)` owned the warmup + measured-iteration loop. `QNNMonitor` does **not**. The iteration loop is now `WinMLSession.perf()`'s caller's responsibility — the monitor only contributes options and parses post-hoc. This is the central architectural shift in the v2.4 PRD.
- **Input generation removed**: `_generate_inputs`, `_ort_type_to_numpy`, `_resolve_shape` are gone. The new contract: the caller (whoever drives `session.run(inputs)` inside the `perf` context) generates inputs. `commands/perf.py` does this — it's not lost, just relocated to where it belongs.
- **`_working_directory` context manager removed**: the old code did `os.chdir(self.output_dir)` to make QNN drop `*_schematic.bin` into the right place. That violates C-5 (no process-global state mutation). The replacement is **post-hoc location**: `_find_schematic()` globs output_dir, then mtime-gates a CWD fallback. This is the FR-12/C-5 enforcement and a genuine behaviour change: if QNN drops the schematic in CWD and the CWD's mtime check fails, the schematic is discarded with `basic_fallback` instead of silently consumed.
- **Owner-enforced provider keys (C-3)**: `profiling_level` and `profiling_file_path` are applied AFTER `dict.update`, so callers cannot override them via `extra_provider_options`. The old `QNNProfiler._build_provider_options` baked everything into a single dict literal — caller-mutability vs. owner-enforcement is a real semantic difference.
- **HTP defaults dropped**: the old code unconditionally set `backend_path=QnnHtp.dll`, `htp_performance_mode=high_performance`, `htp_graph_finalization_optimization_mode=3`, `enable_htp_fp16_precision=1`. The new monitor sets **none** of these. Bundled-ORT users must supply them via `extra_provider_options`; WinML-ORT users get WinML's tuned defaults preserved. **This is a real callsite-level behaviour change for the bundled wheel path.**
- **EP pinning**: `ep_name="qnn"` ClassVar forces `WinMLSession` onto the QNN EP path. Old code passed `providers=["QNNExecutionProvider"]` directly to `InferenceSession`.
- **`requires_session_teardown=True`**: encoded as a class-level contract, consumed by `session.py` at line 732. Old code did `del session` inside `with _working_directory(...)` — equivalent intent, implementation now driven by the session.
- **`int("0" or 0) → round(float(...))` fix (the called-out hardening)**: the metadata extraction previously did `meta.get("accel_execute_cycles", 0)` and used the value directly. The new path forces `round(float(val))`, so float-string inputs (`"12345.6"`) are lossy-rounded rather than dropping the entire trace. The latent bug: with `total_cycles=0` and `accel_us=0`, `cycle_to_us=0.0` and every `OperatorMetrics.duration_us=0.0` — the trace would render as "the entire model ran in 0 µs."
- **Status enum coverage**: `ok`, `no_data` (`_parse_artifacts` CSV-missing path), `parse_failed` (`_parse_artifacts_safe` exception path), `basic_fallback` (`_parse_artifacts` detail-with-QHAS-unavailable path). All four are reached by `tests/unit/session/monitor/test_qnn_monitor.py` (lines 145, 157, 408-435, 482-484). The fifth member of `TraceStatus` — `"not_run"` — is **not** produced by this monitor (it's a placeholder for monitors that haven't run yet; the v2.4 PRD explicitly removes the `not_run` envelope from this code path).
- **Empty-string `op_type` no longer truthy**: `_resolve_op_type` uses `if mapped:` so a defensive `""` in the ONNX map falls through to L2/L3/L4 instead of short-circuiting with `""`.
- **`OperatorMetrics.op_path` vs `name` semantics fixed**: the old `QNNProfiler._from_csv` set both `name` and `op_path` to `op["name"]` (the CSV event-id, not a true path). The new code keeps `name` as the resolved op-type (`"Conv"`) and `op_path` as the framework path. This is a schema-level correctness improvement consumed by report writers.

## Risks / subtleties
- **`_resolve_op_type` was verified**: walked manually against the file. L1 truthy check on `mapped` correctly drops empty string; L2 truthy check correctly drops empty `ep_authoritative`; L3/L4 collapsed into `self._heuristic_op_type(op_path) or op_path` — `_heuristic_op_type` already returns `cleaned` rather than `""` for trailing-slash inputs, so the final `or op_path` is only reachable if `op_path` itself is empty or whitespace-only. The chain is correctly wired. **Headline confirmation**: FR-14 satisfied.
- **CSV path calls resolver with `ep_authoritative=None`**: L2 is structurally unreachable from the basic-CSV codepath. This is fine — the CSV format genuinely has no authoritative op-type column. QHAS path passes `op["name"]` (QHAS-authoritative `qnn_op_type`) as L2.
- **Vocabulary mismatch across paths**: CSV path produces ONNX op symbols at L3 (`"Conv"`, `"Add"`); QHAS path produces QNN symbols at L2 (`"Conv2d"`, `"ElementWiseAdd"`). The ONNX-primary L1 lookup is what reconciles them. When the ONNX map is unavailable, basic and detail mode produce different `name` columns for the same model. Surfaced loud and clear in the `_try_qhas` docstring.
- **`_LEVEL_TO_PROFILING["basic"] = "detailed"`**: anyone debugging at the ORT level needs the map in hand to make sense of `profiling_level="detailed"` traces being labelled "basic" in the report.
- **R-2 retry is a single 50ms sleep**: if Windows file-handle flush takes longer than 50ms under load, the monitor returns `no_data`. Tunable point.
- **`_find_schematic` 5s mtime tolerance**: arbitrary; on a heavily clock-skewed filesystem this could either reject a genuine artifact or accept a stale one. Documented but not parameterised.
- **`_try_qhas` swallows all exceptions to `(None, None, None)`**: the only signal that QHAS failed is the `basic_fallback` status — caller must check it.
- **`num_samples` integer parse is unhardened**: `int(meta.get("num_samples", 0) or 0)` at line 511 still uses bare `int(...)`. If QNN ever emits `"5.0"` for this field, it raises `ValueError`, the surrounding `_parse_artifacts_safe` catches it, and the result becomes `parse_failed`. The `_to_int` hardening was not propagated here. Worth a follow-up.
- **`parse_existing_artifacts` reach-around**: the constructor pins `_csv_path = <output_dir>/profiling_output.csv`, then `parse_existing_artifacts` immediately overwrites `instance._csv_path = csv_path.resolve()` — a private-attribute reach-around documented in the code as "honour the caller's explicit path." A `csv_filename` constructor argument would be cleaner.

## Open questions / TODOs surfaced
- **No tempdir cleanup on `output_dir=None`**: long-running services that exercise this path will leak `qnn_profile_*` directories under `%TEMP%`. Should there be a max-N-keep policy or an explicit `cleanup()` method?
- **`_to_int` defaults silently to 0**: a corrupted SDK output produces an `ok`-status result with zeroed metadata (and thus `cycle_to_us=0`, and thus every `duration_us=0`). Consider escalating to `status="parse_failed"` when both metadata fields fail to parse — the current behaviour silently produces a "0 µs everywhere" trace.
- **Lazy-load contract for `is_available()`**: the inline `TODO(qnn_monitor)` flags that if `ort.get_ep_devices()` ever stops surfacing the QNN device pre-DLL-load, this probe will need to switch to calling `register_ep` on the matching `EPEntry` directly. The check currently relies on filesystem discovery being sufficient for device visibility — an undocumented contract of the WinML EP registry.
- **`num_samples` parse-hardening gap**: `_to_int` was applied to two fields, not all three numeric metadata fields. Inconsistent.
- **HTP defaults are now caller responsibility on bundled-ORT path**: nothing in the codebase reminds bundled-ORT users to pass `backend_path` etc via `extra_provider_options`. A factory like `QNNMonitor.for_bundled_ort(level)` would be the obvious clean shape.

## Simplification opportunities
The file is well-decomposed but a few helpers warrant inlining or merging:

- **`_parse_artifacts_safe` vs `_parse_artifacts`**: the safe wrapper is a 4-line try/except over a single call site that exists for documentation value (single source of truth for the parse-failure contract). It IS called from two places (`__exit__` and `parse_existing_artifacts`), so it earns its keep — but consider folding `_make_failure_result` into the `except` branch since `_make_failure_result` is the **only** thing inside `_parse_artifacts_safe`'s exception arm. Currently `_make_failure_result` is called from two places: the `except` arm and the CSV-missing path. Worth keeping as a helper.
- **`_heuristic_op_type` has a single caller**: `_resolve_op_type`. The chain `mapped or ep_authoritative or self._heuristic_op_type(op_path) or op_path` could be written as a single conditional expression; splitting it out earns its keep purely as a docstring anchor for the spec §3.2 Phase 0 fix. Borderline.
- **`_to_int` is a closure inside `_parse_artifacts`**: there are exactly two call sites, both within the same method. The closure-form is the right call — promoting it to a module-level or class-level helper would just spread the logger naming concern. Leave it.
- **`output_dir` property is a one-liner over `self._output_dir`**: the only reason for the property is the docstring (the "never auto-cleaned" warning). Acceptable.
- **`is_available()` two-path probe could be split**: at 33 lines with two distinct `try` blocks, it's at the boundary of "should split into `_check_bundled_ort()` and `_check_winml()`." Currently fine since both paths are short, but if a third probe path is ever added (e.g. a cloud-managed runtime), splitting becomes mandatory.
- **`_try_qhas` is 60 lines doing two distinct things**: artifact location (qnn_log, schematic, SDK, viewer-shell-out) and JSON parse. The artifact-location half is the live path; the offline path skips it entirely. Could pull the live-path artifact gathering into a `_locate_qhas_inputs()` private helper. Borderline — currently each branch is short enough that the `if qhas_override is not None: ... else: ...` shape reads clearly.
- **Inline `_make_failure_result` into `_parse_artifacts_safe`**: tempting since the exception arm has only one job. Reject — `_parse_artifacts` itself also returns `_make_failure_result(status="no_data", ...)` on the CSV-missing path, so the helper has two genuine callers.
- **`set_onnx_op_types` is a defensive-copy one-liner**: documented purely for the override-the-no-op-base-class semantics. Keep.

Net: the file is at the natural size for what it does. The major decomposition (CSV/QHAS parsers in `qnn._internal`, viewer shell-out in `qnn.viewer`) already happened — the visible 657 lines are the monitor's irreducible orchestration surface.
