# src/winml/modelkit/session/monitor/qnn/_internal.py

## TL;DR
NEW private parser module (+443 lines, no removals at this path) that absorbs the two deleted public-ish files `src/winml/modelkit/optracing/qnn/csv_parser.py` (-227) and `src/winml/modelkit/optracing/qnn/qhas_parser.py` (-113) into a single underscore-prefixed module under `session/monitor/qnn/`. **The move is NOT a pure rename**: three substantive behavior changes were folded in — (1) `_extract_summary` renames every raw QHAS source key to the renderer's user-facing vocabulary (fixes the I-9 disjoint-key bug); (2) `_transform_op` now uses `qnn_op_type` as `name` (was `qnn_op`) and drops the redundant `op_type` field; (3) every bare `dict[key]` access on the QHAS side is routed through a new `_require(d, key, context)` helper so SDK schema drift names the missing field in the log. The CSV side additionally retains a `samples_cycles` per-op list for downstream p90 derivation, populates `op_path` separately from `name` in `_parse_node_event` (via the new `_split_op_event_id` helper) and strips `_token_*` suffixes before splitting. `_TOKEN_SUFFIX` is now imported by `qnn_monitor.QNNMonitor._heuristic_op_type` (the CLAUDE.md `_`-name exception) so CSV-path and resolver-path strip semantics stay in lock-step.

## Role before vs after
- **Before**: Two separately importable modules under `optracing/qnn/`. `csv_parser.parse_qnn_profiling_csv` returned dicts with `{name, op_id, cycles}` per op (no `op_path`, no per-sample list). `qhas_parser.parse_qhas` returned summary dicts keyed by raw QHAS field names (`time_us`, `total_dram_read`, ...) that the renderer never read.
- **After**: One private module under `session/monitor/qnn/`. Both parsers return enriched dicts: CSV ops carry `{name, op_path, op_id, cycles, samples_cycles}`; QHAS summary uses renderer-facing keys (`inference_us`, `dram_read_bytes`, ...); QHAS ops use `qnn_op_type` for `name` and a token-stripped `qnn_op` for `op_path`. Only `qnn_monitor.py` is allowed to import non-`_` names per the architecture regression test at `tests/unit/architecture/test_qnn_imports.py` (PRD NFR-8).

## Symbol-level changes
Module-level constants (both unchanged-in-text from `csv_parser.py`):
- `_OP_PATTERN = re.compile(r"(.+?)(?:\s+|:)OpId_(\d+)\s*\(cycles\)")` — Event Identifier parser.
- `_TOKEN_SUFFIX = re.compile(r"_token_\d+(?:_\d+)?")` — now also imported by `qnn_monitor.QNNMonitor._heuristic_op_type` (sibling production module reaching for a `_`-prefixed name; unusual but documented in this module's comment and in `__init__.py`).

CSV-side (basic-mode profiling):
- `_split_op_event_id(event_id) -> (op_type, op_path)` — **NEW**, did not exist in deleted `csv_parser.py`. Pulls leaf op-type out of slash-delimited paths; the legacy `_parse_node_event` produced only `{name, op_id, cycles}` and never separated path from type.
- `parse_qnn_profiling_csv(csv_path) -> dict` (public, re-exported via `qnn/__init__.py`) — same outer signature; returned dict now carries `operators` entries with `op_path` and `samples_cycles` fields.
- `_read_csv`, `_extract_metadata`, `_extract_samples` — unchanged.
- `_parse_node_event` — **modified**: now returns `{"name": name, "op_path": op_path, "op_id": op_id, "cycles": cycles}` (was `{"name", "op_id", "cycles"}`). Strip-then-split order: `_TOKEN_SUFFIX.sub("", raw_name)` first, then `_split_op_event_id(cleaned)`.
- `_aggregate_operators` — **modified**: maintains `samples_cycles: dict[int, list[int]]`; output dicts gain a per-op `samples_cycles` list. Caller (`QNNMonitor._parse_artifacts`) converts to µs via the ROOT-derived `cycle_to_us`. Comment explicitly justifies leaving conversion to the caller because the ratio lives in ROOT metadata.

QHAS-side (detail mode):
- `_require(d: dict, key: str, context: str) -> Any` — **NEW**. Raises `KeyError(f"Required QHAS field {key!r} is missing in {context}")`.
- `parse_qhas(qhas_data) -> dict` (public, re-exported). Outer signature unchanged.
- `_extract_summary(data) -> dict` — **substantively modified**: renames 13 of 14 keys to renderer vocabulary. Per the docstring this fixes the pre-Bundle-A I-9 bug where 5/6 user-facing keys were disjoint between parser and renderer, silently rendering the detail summary line empty for real production data. Pinned by `test_extract_summary_produces_renderer_expected_keys` in `tests/unit/session/monitor/qnn/test_qhas_parser.py`.
- `_transform_op(op, cycle_to_us) -> dict` — **substantively modified**: `name` is now `op["qnn_op_type"]` (was `op["qnn_op"]`); `op_path` is `_TOKEN_SUFFIX.sub("", op["qnn_op"])` (was raw `op["qnn_op"]`); the redundant `"op_type"` key is dropped from the returned dict.
- `_vtcm_ratio(op) -> float | None` — unchanged.

## Behavior / contract changes
1. **`_extract_summary` key renaming (I-9 fix)** — the renderer at `monitor/report.py::_display_detail_report` reads `inference_us`/`execute_us`/`dram_read_bytes`/`dram_write_bytes`/`vtcm_peak_bytes`/`utilization_pct`. The old `qhas_parser.py` produced `time_us`/`graph_execute_us`/`total_dram_read`/`total_dram_write`/`peak_vtcm_alloc`/`utilization_pct`. Only `utilization_pct` matched; the detail summary line rendered empty silently. The new parser does the rename inside `_extract_summary` and pins the contract via a snapshot test.
2. **`_transform_op` op-type field semantics** — old: `name = qnn_op` (framework path), `op_path = qnn_op`, plus a third `op_type = qnn_op_type` field. New: `name = qnn_op_type` (authoritative QNN op type like `"Conv2d"`), `op_path = _TOKEN_SUFFIX.sub("", qnn_op)` (token-stripped framework path), no separate `op_type` field. The comment explains this is the L2 layer of the FR-14 fallback chain: `QNNMonitor._resolve_op_type` reads `op["name"]` and treats it as `ep_authoritative`.
3. **`_TOKEN_SUFFIX` strip on QHAS `op_path`** — without this strip, FR-14 L1 ONNX-primary lookup is silently inert in detail mode: production map keys are clean (`/encoder/conv1/Conv`) but QHAS `qnn_op` carries `_token_1_2` suffixes. The strip is idempotent on already-clean strings; the in-code comment explicitly calls this out as a correctness fix.
4. **`_split_op_event_id` separation of `name` vs `op_path`** — the CSV path's `_parse_node_event` previously stuffed the entire raw event id (path-form or bare) into a single `name` field. The new helper splits leaf-as-type from full-path-as-path so `OperatorMetrics.name` and `.op_path` carry distinct concepts as the dataclass contract requires. Pinned by `test_event_id_split.py`.
5. **`samples_cycles` retention** — `_aggregate_operators` now returns a `samples_cycles` per-op list so `QNNMonitor` can compute p90 / total / count downstream. Old aggregator returned average only.
6. **Named-`KeyError` schema-drift hardening** — `_try_qhas` in `qnn_monitor.py` catches `except Exception` and logs `"QHAS JSON parse failed: %s"`. With bare `d[k]` the log carried just `'time_us'`; with `_require` it carries the structural context (`"Required QHAS field 'time_us' is missing in htp_overall_summary row"`).

## Cross-file impact
- `src/winml/modelkit/session/monitor/qnn/__init__.py` re-exports `parse_qhas`, `parse_qnn_profiling_csv`; `__all__` lists exactly these two names. Direct imports of `_internal` from outside `qnn_monitor.py` are caught by `tests/unit/architecture/test_qnn_imports.py`.
- `src/winml/modelkit/session/monitor/qnn_monitor.py` imports `_TOKEN_SUFFIX, parse_qhas, parse_qnn_profiling_csv` from `.qnn._internal`. The `_TOKEN_SUFFIX` import is the documented sibling-production exception; `_heuristic_op_type` uses it to mirror `_parse_node_event` strip semantics.
- `qnn_monitor._parse_artifacts` calls `parse_qnn_profiling_csv`, reads `parsed["operators"][i]["op_path"]` and `["samples_cycles"]` (both NEW fields). The local `_to_int(val, field)` helper in `qnn_monitor.py` accepts float-string metadata values (e.g. `"12345.6"`) — this hardening lives on the consumer side, NOT in `_internal.py` (the parser keeps raw `row.get("Time", "").strip()` and a bare `int(time_val)` in `_extract_metadata` and `_parse_node_event` — both will raise `ValueError` on a float string).
- `qnn_monitor._try_qhas` reads `op["name"]` from each QHAS-parsed op and passes it as `ep_authoritative` to `_resolve_op_type`. This load-bears on the new `name = qnn_op_type` semantics.
- `monitor/report.py::_display_detail_report` consumes the renamed summary keys.

## Risks / subtleties
- **Lost in the move — verified absent**: The deleted `qhas_parser._transform_op` emitted three fields `{name, op_path, op_type}` from the same source `qnn_op`/`qnn_op_type`. The new module emits only `{name, op_path}` — the `op_type` field is dropped entirely. This is intentional (the resolver chain in `QNNMonitor._resolve_op_type` only needs `name` as `ep_authoritative`), but any out-of-tree consumer of the old `qhas_parser` that read `op["op_type"]` directly would now get `KeyError`. No internal consumer references it, but worth flagging.
- **`_require` raises `KeyError`, not a typed exception**. `_try_qhas` catches `Exception`, so the named message survives, but downstream code wanting to disambiguate "schema drift" from "caller bug" cannot do so.
- **`_extract_metadata` and `_parse_node_event` still use bare `int(time_val)`** — if QNN SDK ever emits a float string in ROOT cycles/us or NODE cycles, this raises `ValueError` and the row drops (CSV path) or the whole metadata block returns zeros via the `or 0` masks (CSV ROOT path). The hardening in `qnn_monitor._to_int(val, field)` only protects the metadata consumed there, not the parser-internal `int(...)` calls.
- **`_TOKEN_SUFFIX` shared with `qnn_monitor`** — module-scope regex; safe across threads. The sibling import of `_TOKEN_SUFFIX` is unusual in a `_`-prefixed convention; it's documented in this module's comment and the architecture test's exception list. If someone moves the regex into a public symbol later, the architecture test allow-list needs to relax.
- **`_extract_metadata` "first occurrence wins"** — for multi-sample CSVs the rule silently discards per-sample metadata variance. Documented in the docstring; matches legacy behavior.
- **`_aggregate_operators` arithmetic mean across samples** — small-sample outliers skew it. `samples_cycles` is exposed downstream for callers wanting robust statistics.
- **`_split_op_event_id` returns `("", "")` for whitespace-only input but `(s, s)` for trailing-slash input** — asymmetric edge handling pinned by `test_event_id_split.py::test_trailing_slash_falls_back_gracefully`.

## Open questions / TODOs
- The deleted `qhas_parser.py` produced a third `op_type` field that the new module drops. If any test fixture or external doc referenced it the silent drop is invisible until something explicit breaks — recommend a quick `Grep "op_type" tests/unit/session/monitor/qnn/` audit.
- `_require(d: dict, ...)` is typed as `dict`, not `Mapping` — strict callers passing dict-likes lint-warn.
- The QHAS-key → renderer-key renaming is one-way and undocumented at the renderer side; introspecting raw QHAS requires knowing both vocabularies.
- The bare `int(time_val)` calls in `_extract_metadata` and `_parse_node_event` could adopt the same `round(float(...))` defensive idiom that `qnn_monitor._to_int` uses, eliminating the two-place hardening asymmetry.

## Simplification opportunities
- **`_extract_summary` is 14 nearly-identical `_require(raw, "<source>", ctx)` calls**. A small `_RENAME_MAP: dict[str, str]` (renderer-key → source-key) plus a single comprehension `{rk: _require(raw, sk, ctx) for rk, sk in _RENAME_MAP.items()}` would halve the line count and surface the rename contract as data, not control flow. The 14 individually-named entries also make it easy to silently drop one in a future edit.
- **`_split_op_event_id` is only called from `_parse_node_event`** (verified — sole caller in the file). It could fold into `_parse_node_event` as a 2-line inline, but the docstring carries the "heuristic-only, never reuse on QHAS path" warning that is itself load-bearing — pinning it to a named helper makes the warning discoverable to `Grep` (and `test_event_id_split.py::test_docstring_marks_helper_as_heuristic_only` pins the wording). Keep the function for the warning surface; collapse the body if needed.
- **`_aggregate_operators` keeps three parallel dicts** (`totals`, `counts`, `samples_cycles`) keyed by `op_id`. A single `dict[int, _Accumulator]` (small frozen dataclass with `cycles_total: int`, `count: int`, `samples: list[int]`, `name: str`, `op_path: str`) would reduce the keying noise and let the average expression read straight off the accumulator. Same complexity, fewer lookups.
- **`_TOKEN_SUFFIX.sub("", _require(op, "qnn_op", ctx))` in `_transform_op`** computes the stripped path inline; a `_clean_op_path(raw: str) -> str` helper would localise the strip rule and make it trivially diff-able if QNN ever introduces a second compiler suffix shape.
- **The `_to_int` hardening lives in `qnn_monitor.py` but the bare `int(time_val)` in `_parse_node_event` is the matching exposure**. Move `_to_int` into `_internal.py` as `_safe_int` and use it on both sides; the consumer module is the wrong layer for SDK-output hardening because it has to bypass the parser to apply it.
- **Drop the `_internal.py` filename if `qnn/__init__.py` re-exports cleanly** — the architecture regression test enforces the boundary by AST scan, not by filename. A flatter `qnn/parsers.py` with `_`-prefixed helpers reads the same. The current name is fine; just noting the boundary is enforced by the test, not the convention.
