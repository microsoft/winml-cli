1 · Problem Statement

The autoconfig sweep loop produces unreliable conclusions because of performance measurement noise. Even with the Phase A/B/C protocol, results vary run-to-run — leading to false promotions into the knowledge base and wasted sweep time on un-reproducible hypotheses.

2 · Confidence-Gated Promotion (L1 → L5)

Findings are promoted through confidence levels. The KB only contains L3+ entries. Each level has a quantitative gate — no manual judgement required.

L1
Observed — Single Model, Single Run
A hypothesis beats baseline in one model sweep. Stored in results.json only.
Gate: median gain > 5%. No range check.
↓
Paired A/B bench → range non-overlap criterion
L2
Confirmed — Single Model, Statistically Robust
All (N_full + N_confirm) p50s beat ALL baseline p50s (ranges don't overlap). 95% CI on gain excludes 0.
Gate: max(hyp_p50s) < min(baseline_p50s). Stored as L2 candidate in results.json.
↓
promote_findings.py — same flags confirmed on 2+ models of same arch class
L3
Generalized — Architecture-Level Rule
Same optimization flags produce L2-quality gains on ≥2 models of the same architecture class (e.g., pure-ViT, ResNet, BERT). Automatically written to ep_knowledge/<ep>.json.
Gate: ≥2 L2 confirmations with same (flags, arch_class). Finding scope: "applies_to_arch".
↓
promote_findings.py — confirmed across architecture classes
L4
Cross-Cutting Rule
Same optimization applies across ≥3 different model architecture classes. The finding's applies_to scope broadens from arch-specific to EP-wide (e.g., "all transformer models on QNN NPU").
Gate: ≥3 L2 confirmations across ≥3 different arch_class values.
↓
analyze_insight.py predicts outcome from graph fingerprint alone
L5
Predictive — No Sweep Required
The graph fingerprint (from analyze_insight.py) predicts whether an optimization will help or hurt before running. The sweep loop skips this hypothesis and directly generates the optimal config.
Gate: L4 rule + graph analysis pattern match with <5% false-positive rate on held-out models.
Current state (2026-06-18): Most findings are manually promoted at L2. npu-001 and npu-007 have enough multi-model evidence to be L3. The L1→L5 promotion pipeline (promote_findings.py) does not yet exist. L5 prediction is not yet implemented.

3 · Fix #1 — Paired A/B Bench Protocol

❌ Current: Sequential (Biased)

All baseline sessions → then all hypothesis sessions. Baseline runs when device is cold; hypothesis runs when device is warm. The "gain" includes thermal drift, not just optimization benefit.

# Sessions over time (device heats up →)
h0: [base_1] [base_2] [base_3]
h6: [hyp_1]  [hyp_2]  [hyp_3]
# baseline measured at T=cool, hyp at T=warm
# delta = optimization + thermal drift (confounded)
✅ New: Paired A/B (Unbiased)

For each pair: run baseline, then hypothesis, in the same thermal window. Take the within-pair ratio. Average ratios across N pairs. Thermal drift cancels out.

# Each pair is one thermal window
pair_1: [base] → [hyp]   # ratio_1 = (base-hyp)/base
pair_2: [base] → [hyp]   # ratio_2
pair_3: [base] → [hyp]   # ratio_3
gain = mean(ratio_1..3) ± 95% CI
# thermal drift appears equally in base & hyp → cancels
def paired_ab_bench(
    baseline_path: Path, hyp_path: Path,
    n_pairs: int = 3, iters: int = 500, cool_down_s: float = 30
) -> dict:
    """Interleaved A/B bench. Returns gains_pct list + CI."""
    gains = []
    for i in range(n_pairs):
        base_p50 = run_perf_session(baseline_path, iters)
        hyp_p50  = run_perf_session(hyp_path,      iters)
        if base_p50 and hyp_p50:
            gains.append((base_p50 - hyp_p50) / base_p50 * 100)
        if i < n_pairs - 1:
            time.sleep(cool_down_s)
    if not gains:
        return {"verdict": "BENCH_FAIL"}
    mean_gain = statistics.mean(gains)
    ci_half   = 1.96 * statistics.stdev(gains) / math.sqrt(len(gains)) if len(gains) > 1 else 999
    return {
        "gains_pct":   gains,
        "mean_gain_pct": round(mean_gain, 2),
        "ci_half_95":  round(ci_half, 2),
        "verdict": (
            "KEEP_CONFIRMED"  if mean_gain - ci_half > 5.0   else
            "DISCARD"         if mean_gain + ci_half < -2.0  else
            "MARGINAL"
        ),
    }

4 · Fix #2 — Adaptive n_sessions

Instead of always running exactly N pairs, keep sampling until the 95% confidence interval is tight enough to make a decision — or the budget runs out.

Stopping Criterion

Stop early when: CI_lower > +5% (definitely KEEP) or CI_upper < -2% (definitely DISCARD).

Force stop at MAX_PAIRS = 8 regardless. Report MARGINAL with uncertainty flag.

Benefit: stable models finish in 3 pairs; noisy models (MobileViT DVFS) automatically get more samples.

Budget Allocation

Priority queue: test highest-prior hypotheses first (from architecture pruning). If a KEEP_CONFIRMED is found early, reduce budget for remaining hypotheses.

E.g., if h6 (matmul_transpose) is KEEP_CONFIRMED at pair 3, remaining hypotheses get only 2 pairs each (quick reject/confirm).

def get_hypothesis_skip_set(model_type: str, candidates: list[FusionCandidate]) -> set[str]:
    """Return hypothesis IDs to skip based on architecture fingerprint + KB rules."""
    skip = set()
    tags = {c.tag for c in candidates}

    # npu-006: conv fusions → FusedConv → QNN CPU fallback
    if "conv_dense" in tags:
        skip |= {"h4", "h5"}

    # npu-010 / gpu-008: highdimRTR harmful on CNN-ViT hybrid
    if "gemm_reshape_transpose_unfold" in tags:
        skip.add("h9")  # highdimRTR

    # cpu-008: layer_norm_fusion harmful on CNN-ViT hybrid
    if model_type == "mobilevit" and "ep" == "cpu":
        skip.add("h6")   # layer_norm_fusion

    # cpu-001: opset19/21 harmful on dense-Transpose models (CPU only)
    if "dense_transpose" in tags and "ep" == "cpu":
        skip |= {"h2", "h3"}  # opset19, opset21

    return skip

6 · Fix #4 — promote_findings.py

A standalone post-processing script that runs after sweeps complete. Reads all results.json files, applies the confidence ladder, and auto-updates the KB.

# promote_findings.py — core logic sketch

def collect_l2_candidates(sweep_dirs: list[Path]) -> list[dict]:
    """L1→L2: range non-overlap confirmed for a single model."""
    candidates = []
    for d in sweep_dirs:
        r = json.loads((d / "results.json").read_text())
        baseline_p50s = r["hypotheses"]["h0"]["full"]["p50s_ms"]
        for h_id, h in r["hypotheses"].items():
            if h.get("verdict") == "KEEP_CONFIRMED":
                hyp_p50s = h["full"]["p50s_ms"]
                if max(hyp_p50s) < min(baseline_p50s):  # L2 gate
                    candidates.append({
                        "model_id": r["model_id"], "arch": r["model_type"],
                        "ep": r["ep"], "h_id": h_id,
                        "flags": h.get("extra_optim", {}),
                        "gain_pct": h["mean_gain_pct"],
                    })
    return candidates

def promote_to_l3(l2s: list[dict]) -> list[dict]:
    """L2→L3: same flags confirmed on ≥2 models of same arch class."""
    grouped = defaultdict(list)
    for c in l2s:
        key = (c["ep"], frozenset(c["flags"].items()), c["arch"])
        grouped[key].append(c)
    return [
        {"ep": ep, "flags": dict(flags), "arch": arch,
         "evidence": g, "mean_gain": statistics.mean(x["gain_pct"] for x in g)}
        for (ep, flags, arch), g in grouped.items() if len(g) >= 2  # L3 gate
    ]

7 · Fix #5 — Thermal Reference Model (P2)

Concept

Before each sweep session, run a fixed tiny model (e.g., a 3-layer MLP exported to ONNX) for 100 iters. Its latency is a proxy for current device thermal state.

Store as thermal_ref_p50_ms in each session result. Post-hoc: normalize per-session gains by thermal_ref to make cross-day / cross-run comparisons valid.

When to Use

Cool enough (thermal_ref ≤ 1.05 × cold_baseline): proceed normally.

Too hot (thermal_ref > 1.3 × cold_baseline): wait 60s and retry. Max 3 retries, then flag session as "HOT_RUN".

HOT_RUN sessions are excluded from L2 promotion but kept for informational display.
